# Функциональность системы

## 1. Объединение спектральных каналов

**Что делает:** Объединяет несколько одноканальных TIFF файлов в один многоканальный файл.

**Как работает:**
- Пользователь загружает отдельные спектральные каналы (B01, B02, B03 и т.д.)
- Система автоматически определяет названия каналов по имени файла
- Можно изменить порядок каналов перетаскиванием
- Каналы объединяются в правильном порядке
- Создаётся многоканальный TIFF файл

**Технические детали:**
- Backend: `/api/stack/bands` (POST)
- Сохраняется в `/data/results/{id}/stacked.tif`
- Поддержка Sentinel-2 (12 каналов) и Landsat-8 (7 каналов)
- Автоопределение по паттернам: B01, B02, _B1_, _B2_ и т.д.

**Файлы:**
- `backend/app_aplication/api/stack.py`
- `frontend/app/components/BandStackingModal.tsx`

## 2. Расчёт индексов

**Что делает:** Рассчитывает вегетационные и водные индексы из многоканальных снимков.

**Поддерживаемые индексы:**
- NDVI (Normalized Difference Vegetation Index)
- NDWI (Normalized Difference Water Index)
- EVI (Enhanced Vegetation Index)
- NBR (Normalized Burn Ratio)
- GNDVI (Green NDVI)
- GEMI (Global Environment Monitoring Index)

**Как работает:**
1. Загружается многоканальный TIFF
2. Система извлекает нужные каналы (Red, NIR, Green и т.д.)
3. Применяется формула индекса
4. Результат нормализуется в диапазон 0-255
5. Сохраняется как одноканальный TIFF + превью PNG

**Технические детали:**
- Backend: `/api/index/calculate` (POST)
- Сохраняется `source.tif` (исходник) и `result.tif` (результат)
- Поддержка 5 спутников: Sentinel-2, Landsat-8, Канопус-В, Метеор-М, Ресурс-П

**Файлы:**
- `backend/app_aplication/api/index.py`
- `backend/app_aplication/index_calculator.py`

## 3. Выделение водных ресурсов (нейросеть)

**Что делает:** Использует нейронную сеть UNet для сегментации водных объектов.

**Требования:**
- **Ровно 12 каналов Sentinel-2** (B01-B12)
- PyTorch установлен

**Модели:**
- NDWI (Normalized Difference Water Index)
- MNDWI (Modified NDWI)

**Как работает:**
1. Загружается 12-канальный файл
2. Данные подаются в нейросеть UNet
3. Модель выдаёт вероятность воды для каждого пикселя
4. Применяется порог (threshold) для бинаризации
5. Создаётся маска воды (0 = суша, 255 = вода)

**Технические детали:**
- Backend: `/api/water/calculate` (POST)
- Архитектура: UNet с 12 входными каналами
- Модели хранятся в `backend/app_aplication/data/models/`
- Сохраняется `source.tif` и `result.tif` (маска)

**Предупреждения:**
- Система показывает предупреждение если каналов < 12
- В BandStackingModal отображается статус совместимости

**Файлы:**
- `backend/app_aplication/api/water.py`
- `backend/app_aplication/api/unet.py`

## 4. Карта риска

**Что делает:** Создаёт интегральную карту риска на основе нескольких индексов.

**Сценарии:**
- Flooding (затопление)
- Drought (засуха)
- Fire (пожар)

**Как работает:**
1. Рассчитываются индексы: NDVI, NDWI, BSI
2. Применяются веса для выбранного сценария
3. Создаётся интегральная карта риска
4. Применяется нормализация (linear/minmax)
5. Классификация: безопасно (зелёный), средний риск (жёлтый), опасно (красный)

**Технические детали:**
- Backend: `/api/risk/assess` (POST)
- Пороги настраиваются: safe_threshold, danger_threshold
- Сохраняются отдельные индексы в `/indices/`

**Файлы:**
- `backend/app_aplication/api/risk.py`

## 5. Временные ряды

**Что делает:** Позволяет создавать и просматривать временные ряды снимков.

**Функции:**
- Создание временного ряда (название, спутник, индекс)
- Добавление снимков с датами
- Просмотр с интерактивным слайдером
- Автопроигрывание с регулировкой скорости
- Горячие клавиши (←, →, Пробел, Home, End)
- Статистика по каждому снимку

**Как работает:**
1. Создаётся временной ряд с параметрами
2. Добавляются снимки (готовый файл или объединение каналов)
3. Для каждого снимка рассчитывается индекс
4. Снимки сортируются по дате
5. Просмотр в интерактивном режиме

**Хранение:**
- localStorage (для дипломного проекта достаточно)
- Структура: массив серий, каждая содержит массив снимков

**Файлы:**
- `frontend/app/time-series/page.tsx` - список серий
- `frontend/app/time-series/[id]/page.tsx` - просмотр
- `frontend/app/time-series/components/TimeSeriesViewer.tsx`

## 6. История расчётов

**Что делает:** Сохраняет все расчёты и позволяет повторно использовать файлы.

**Функции:**
- Автоматическое сохранение всех расчётов
- Просмотр истории с фильтрацией
- Выбор файла из истории для нового расчёта
- Отображение типа расчёта, параметров, даты

**Как работает:**
1. При каждом расчёте создаётся запись в localStorage
2. Сохраняется: ID, тип, параметры, timestamp
3. Кнопка "Из истории" открывает модальное окно
4. Пользователь выбирает расчёт
5. Загружается `source.tif` (исходный многоканальный файл)
6. Файл готов для нового расчёта

**Важно:**
- Сохраняется `source.tif` - исходный файл, не результат
- Это позволяет использовать один файл для разных расчётов
- Например: объединить 12 каналов → рассчитать NDVI → выбрать из истории → рассчитать воду

**Файлы:**
- `frontend/app/components/HistoryFileSelector.tsx`
- `frontend/app/result/components/HistorySidebar.tsx`

## 7. Сохранение исходных файлов

**Что делает:** При каждом расчёте сохраняет исходный многоканальный файл.

**Структура результата:**
```
/data/results/{id}/
  ├── source.tif      # Исходный многоканальный файл
  ├── result.tif      # Результат расчёта (одноканальный)
  ├── preview.png     # Превью результата
  ├── meta.json       # Метаданные
  └── indices/        # Отдельные индексы (для риска)
```

**Зачем:**
- Повторное использование файла для других расчётов
- Не нужно заново объединять каналы
- Можно рассчитать разные индексы из одного файла

**Реализация:**
- `index.py`: сохраняет загруженный файл как `source.tif`
- `water.py`: сохраняет загруженный файл как `source.tif`
- `risk.py`: сохраняет загруженный файл как `source.tif`
- `calculate-from-stacked`: копирует stacked файл как `source.tif`

## Workflow примеры

### Пример 1: Объединение и расчёт индекса
1. Загрузить 12 отдельных каналов Sentinel-2
2. Объединить через "Объединить отдельные каналы"
3. Выбрать индекс NDVI
4. Рассчитать
5. Результат сохраняется в истории

### Пример 2: Повторное использование
1. Открыть "Из истории"
2. Выбрать предыдущий расчёт NDVI
3. Загружается исходный 12-канальный файл
4. Выбрать режим "Выделение водных ресурсов"
5. Рассчитать маску воды

### Пример 3: Временной ряд
1. Создать временной ряд "Мониторинг озера"
2. Добавить снимок за январь (объединить каналы)
3. Добавить снимок за февраль
4. Добавить снимок за март
5. Просмотреть изменения с помощью слайдера
