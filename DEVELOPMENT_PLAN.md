## Текущее состояние проекта

**Реализовано:**
- Расчёт спутниковых индексов (NDVI, NDWI, BSI и др.)
- Выделение водных объектов (U-Net модели)

- Простой веб-интерфейс (Next.js)
- FastAPI бэкенд
- Локальное хранение истории расчётов (localStorage)

**Архитектура:**
- Frontend: Next.js + TypeScript + Tailwind CSS
- Backend: FastAPI + Python
- Хранение: файловая система (TIFF, PNG, JSON)

## Фаза 1: Добавление базы данных

### Цель
Перейти от файлового хранилища к реляционной БД для хранения метаданных расчётов, пользователей и временных рядов.

### Технологический стек
- **БД:** PostgreSQL (рекомендуется) или SQLite (для простоты)
- **ORM:** SQLAlchemy
- **Миграции:** Alembic

### Схема базы данных

```sql
-- Пользователи
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(255) UNIQUE NOT NULL,
    username VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT NOW(),
    is_active BOOLEAN DEFAULT TRUE
);

-- Расчёты (calculations)
CREATE TABLE calculations (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    filename VARCHAR(255) NOT NULL,
    type VARCHAR(20) NOT NULL, -- 'index', 'water', 'risk'
    
    -- Параметры расчёта (JSON)
    parameters JSONB NOT NULL,
    
    -- Пути к файлам
    result_path VARCHAR(500),
    preview_path VARCHAR(500),
    
    -- Метаданные
    status VARCHAR(20) DEFAULT 'pending', -- 'pending', 'processing', 'completed', 'failed'
    error_message TEXT,
    
    created_at TIMESTAMP DEFAULT NOW(),
    completed_at TIMESTAMP
);

-- Индексы для расчётов
CREATE INDEX idx_calculations_user_id ON calculations(user_id);
CREATE INDEX idx_calculations_type ON calculations(type);
CREATE INDEX idx_calculations_created_at ON calculations(created_at);

-- Временные ряды
CREATE TABLE time_series (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    name VARCHAR(255) NOT NULL,
    description TEXT,
    
    -- Область интереса (GeoJSON)
    area_of_interest JSONB,
    
    -- Параметры анализа
    index_type VARCHAR(50) NOT NULL, -- 'NDVI', 'NDWI', etc.
    satellite VARCHAR(50),
    
    created_at TIMESTAMP DEFAULT NOW()
);

-- Точки временного ряда
CREATE TABLE time_series_points (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    time_series_id UUID REFERENCES time_series(id) ON DELETE CASCADE,
    calculation_id UUID REFERENCES calculations(id) ON DELETE SET NULL,
    
    -- Дата снимка
    acquisition_date DATE NOT NULL,
    
    -- Статистика по области
    mean_value FLOAT,
    min_value FLOAT,
    max_value FLOAT,
    std_value FLOAT,
    
    -- Дополнительные метрики
    metrics JSONB,
    
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_time_series_points_series_id ON time_series_points(time_series_id);
CREATE INDEX idx_time_series_points_date ON time_series_points(acquisition_date);
```

### Задачи

1. **Настройка БД** (2-3 дня)
   - Установка PostgreSQL
   - Настройка SQLAlchemy
   - Создание моделей (models.py)
   - Настройка Alembic для миграций

2. **Миграция данных** (2-3 дня)

   - Сохранение обратной совместимости

3. **Обновление API** (3-4 дня)
   - Изменение эндпоинтов для работы с БД
   - Добавление CRUD операций
   - Обновление логики сохранения результатов

4. **Обновление Frontend** (2-3 дня)

## Фаза 2: Авторизация и управление пользователями

### Цель
Добавить систему регистрации, входа и управления пользователями.

### Технологический стек
- **Аутентификация:** JWT (JSON Web Tokens)
- **Библиотеки:** 
  - `python-jose` для JWT
  - `passlib` для хеширования паролей
  - `python-multipart` для форм
- **Frontend:** NextAuth.js или собственная реализация

### Функциональность

#### Backend

1. **Регистрация** (`POST /api/auth/register`)
   - Валидация email и пароля
   - Хеширование пароля (bcrypt)
   - Создание пользователя в БД
   - Отправка email подтверждения (опционально)

2. **Вход** (`POST /api/auth/login`)
   - Проверка credentials
   - Генерация JWT токена (access + refresh)
   - Возврат токена клиенту

3. **Выход** (`POST /api/auth/logout`)
   - Инвалидация refresh токена

4. **Обновление токена** (`POST /api/auth/refresh`)
   - Проверка refresh токена
   - Выдача нового access токена

5. **Профиль пользователя** (`GET /api/users/me`)
   - Получение информации о текущем пользователе
   - Обновление профиля

#### Frontend

1. **Страницы**
   - `/login` - вход
   - `/register` - регистрация
   - `/profile` - профиль пользователя

2. **Компоненты**
   - `AuthProvider` - контекст авторизации
   - `ProtectedRoute` - защищённые маршруты
   - `LoginForm` - форма входа
   - `RegisterForm` - форма регистрации

3. **Middleware**
   - Проверка токена при каждом запросе
   - Автоматическое обновление токена
   - Редирект на /login при отсутствии авторизации

### Задачи

1. **Backend авторизация** (4-5 дней)
   - Создание auth эндпоинтов
   - JWT middleware
   - Защита существующих эндпоинтов
   - Тестирование

2. **Frontend авторизация** (3-4 дня)
   - Создание страниц login/register
   - AuthContext и хуки
   - Интеграция с API
   - Хранение токенов (httpOnly cookies)

3. **Связывание данных с пользователями** (2-3 дня)
   - Добавление user_id ко всем расчётам
   - Фильтрация данных по пользователю
   - Права доступа

## Фаза 3: Временные ряды и анализ динамики

### Цель
Позволить пользователям создавать временные ряды для отслеживания изменений индексов на выбранной территории.

### Функциональность

#### 1. Создание временного ряда

**UI Flow:**
1. Пользователь загружает несколько снимков одной территории за разные даты
2. Выбирает область интереса (AOI) на карте или загружает GeoJSON
3. Выбирает индекс для анализа (NDVI, NDWI, BSI)
4. Система создаёт временной ряд

**API:**
```python
POST /api/time-series/create
{
  "name": "Мониторинг озера Байкал",
  "description": "NDWI за 2023-2024",
  "index_type": "NDWI",
  "satellite": "Sentinel-2",
  "area_of_interest": {
    "type": "Polygon",
    "coordinates": [...]
  }
}
```

#### 2. Добавление точек во временной ряд

**Процесс:**
1. Пользователь загружает новый снимок
2. Система автоматически рассчитывает индекс
3. Извлекает статистику по AOI (mean, min, max, std)
4. Добавляет точку во временной ряд

**API:**
```python
POST /api/time-series/{series_id}/add-point
{
  "calculation_id": "uuid",
  "acquisition_date": "2024-03-15"
}
```

#### 3. Визуализация временного ряда

**Графики:**
- Линейный график изменения индекса во времени
- Гистограмма распределения значений
- Карта изменений (difference map)
- Тренд и сезонность

**Библиотеки:**
- Frontend: Chart.js или Recharts
- Backend: matplotlib для генерации изображений

**API:**
```python
GET /api/time-series/{series_id}/data
Response:
{
  "series_id": "uuid",
  "name": "...",
  "points": [
    {
      "date": "2024-01-15",
      "mean": 0.45,
      "min": 0.12,
      "max": 0.89,
      "std": 0.15
    },
    ...
  ]
}
```

#### 4. Анализ изменений

**Функции:**
- Расчёт тренда (линейная регрессия)
- Обнаружение аномалий
- Сравнение периодов (до/после)
- Экспорт данных (CSV, JSON)

**API:**
```python
GET /api/time-series/{series_id}/analysis
Response:
{
  "trend": {
    "slope": 0.002,
    "direction": "increasing",
    "r_squared": 0.85
  },
  "anomalies": [
    {
      "date": "2024-02-10",
      "value": 0.12,
      "z_score": -2.5
    }
  ],
  "statistics": {
    "mean": 0.45,
    "std": 0.08,
    "change_percent": 12.5
  }
}
```

### Задачи

1. **Backend - временные ряды** (5-6 дней)
   - CRUD для временных рядов
   - Извлечение статистики по AOI
   - Расчёт трендов и аномалий
   - API для анализа

2. **Frontend - создание и управление** (4-5 дней)

3. **Frontend - визуализация** (5-6 дней)
   - Графики (Chart.js)
   - Интерактивная карта с AOI
   - Таблица данных

## Риски и сложности

### Технические риски

1. **Производительность БД**
   - Большое количество TIFF файлов

4. **Геопространственные данные**
   - Работа с координатами и проекциями

1. **Время разработки**
   - 5-8 недель для одного разработчика
   - Решение: приоритизация функций

2. **Тестирование**
   - Нужны реальные данные для тестирования
   - Решение: создание тестового датасета

3. **Безопасность**
